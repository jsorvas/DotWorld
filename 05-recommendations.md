> Métriques

Taux de Réussite au Premier Passage (First-Run Pass Rate) : Mesure le pourcentage de tests qui passent du premier coup sans nécessiter de correctifs immédiats 

Taux de Régression : Pourcentage de bugs réapparus sur des fonctionnalités déjà validées 

Densité de Défauts : Nombre de bugs rapportés par rapport à la taille du module (ex: bugs/User Story). Cela permet d'identifier les zones "à risque" du logiciel.

Taux d'Automatisation : (Nombre de cas de tests automatisés / Total des cas de tests éligibles) 

Fiabilité des Tests (Flakiness Rate) : Pourcentage de tests qui échouent de manière intermittente sans changement de code
